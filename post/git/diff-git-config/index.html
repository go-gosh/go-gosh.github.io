多个远程仓库, 如何自动设置不同邮箱或用户名

场景：

一些项目提交的时候，需要使用邮箱A，而另一些项目提交的时候需要使用邮箱B。由于项目过多，不可能每次都手动的去git config user.email

解决方案：

  • git > 2.13

我们将需要邮箱A提交的项目，放到~/workspace/code/project_a下面，将需要邮箱B提交的项目放到~/workspace/code/project_b下面。

然后，打开git配置文件~/.gitconfg添加以下内容：

1
2
3
4
5

[includeIf "gitdir:code/project_a"]
    path = ./.gitprojecta

[includeIf "gitdir:code/project_b"]
    path = ./.gitprojectb

并新建~/.gitprojecta文件：

1
2

[user]
    email = aaaaa@aaaa.com

新建~/.gitprojectb文件：

1
2

[user]
    email = bbbbbb@bbbbb.com

这样，在code/project_a目录下的git会使用邮箱A，而code/project_b目录下的git会使用邮箱B，其余git项目则依旧使用global的配置

更多Conditional Includes请查阅官方文档
